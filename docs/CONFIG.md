# cluster.env reference

Every variable in `cluster.env` documented. The file is generated by
`milvus-onprem init`, gitignored, and shared identically across all
peers (distributed automatically by `pair`/`join`).

You can override any variable either at init time (`--flag=value`) or
by editing `cluster.env` directly and running `./milvus-onprem render
&& ./milvus-onprem up`.

---

## Topology

| Variable | Required | Default | Notes |
|---|---|---|---|
| `PEER_IPS` | **Yes** | none | Comma-separated internal IPs of every peer. Must be 1 (standalone) or odd ≥3. Position determines node identity (first IP = node-1). |
| `CLUSTER_NAME` | Yes | `milvus-onprem` | Used as etcd cluster token + MinIO cluster identifier. Same on every peer. |

Each VM figures out its node identity at runtime by matching `hostname -I`
against `PEER_IPS`. If that fails (e.g. unusual NAT setups), set
`FORCE_NODE_INDEX=N` in the environment to override.

---

## Object storage (MinIO)

| Variable | Required | Default | Notes |
|---|---|---|---|
| `MINIO_ACCESS_KEY` | Yes | `minioadmin` | Same on every peer. |
| `MINIO_SECRET_KEY` | Yes | auto-generated by `init` | 24-char base64. Distributed by `pair`/`join`. |
| `MINIO_REGION` | Yes | `us-east-1` | Used for SigV4 signing. Any non-empty value works; must match across peers. |
| `MINIO_DRIVES_PER_NODE` | No | `1` | Drives per node. Most deploys use 1 (one drive per VM). |

⚠ Changing credentials post-deployment requires `teardown --full` and
redeploy. MinIO doesn't support live credential rotation in distributed
mode.

---

## Message queue / WAL

| Variable | Required | Default | Notes |
|---|---|---|---|
| `MQ_TYPE` | Yes | `woodpecker` | `woodpecker` (Milvus 2.6 embedded WAL — recommended) or `pulsar` (not yet implemented in v0). |

For Milvus 2.5 support (planned via community PR), `MQ_TYPE` will
default to `pulsar` and the templates in `templates/2.5/` will deploy
a Pulsar singleton.

## Container image versions

| Variable | Default | Notes |
|---|---|---|
| `MILVUS_IMAGE_TAG` | `v2.6.11` | Pinned. Bumping the patch (e.g. `v2.6.12`) is safe. Bumping major.minor (e.g. `v2.5.4`) routes to a different `templates/X.Y/` directory. |
| `ETCD_IMAGE_TAG` | `v3.5.25` | |
| `MINIO_IMAGE_TAG` | `RELEASE.2024-05-28T17-19-04Z` | |
| `NGINX_IMAGE_TAG` | `1.27-alpine` | |

All four images must be identical across peers. After editing, run
`./milvus-onprem render && ./milvus-onprem up` to roll the changes.

### Supported Milvus versions

| Version | Status |
|---|---|
| `2.6.x` | Fully supported (default, tested) |
| `2.5.x` | Planned via community PR (`templates/2.5/` with Pulsar singleton) |
| `1.x` and earlier | Not supported, won't be |

## Storage

| Variable | Default | Notes |
|---|---|---|
| `DATA_ROOT` | `/data` | Root for all on-disk state. Three subdirs are created: `${DATA_ROOT}/etcd`, `${DATA_ROOT}/minio`, `${DATA_ROOT}/milvus`. |

## Ports

| Variable | Default | Used by |
|---|---|---|
| `ETCD_CLIENT_PORT` | `2379` | etcd gRPC client connections |
| `ETCD_PEER_PORT` | `2380` | etcd Raft inter-member traffic |
| `MINIO_API_PORT` | `9000` | MinIO S3 API |
| `MINIO_CONSOLE_PORT` | `9001` | MinIO web console (optional) |
| `MILVUS_PORT` | `19530` | Milvus gRPC (direct, behind nginx) |
| `MILVUS_HEALTHZ_PORT` | `9091` | Milvus /healthz endpoint |
| `NGINX_LB_PORT` | `19537` | Client-facing entry point |
| `PAIR_PORT` | `19500` | HTTP rendezvous during initial deploy only |

All ports are settable at init time:

```bash
./milvus-onprem init \
  --peer-ips=... \
  --milvus-port=29530 \
  --lb-port=29537 \
  --etcd-client-port=12379
```

Or edit cluster.env directly. After any port change, every peer needs
`render && up` to pick it up.

## Watchdog

| Variable | Required | Default | Notes |
|---|---|---|---|
| `WATCHDOG_MODE` | Yes | `monitor` | `monitor` (alert-only) or `auto` (would fire automatic failover; not yet implemented for N-node since etcd's own Raft handles failover). |
| `WATCHDOG_INTERVAL_S` | No | `5` | Seconds between peer probes. |
| `WATCHDOG_FAILURE_THRESHOLD` | No | `6` | Consecutive failed probes before alerting. Default = 30s window. |

In v0, the watchdog is alert-only: it logs to journal when a peer is
unreachable. With proper N-node Raft quorum, the *cluster* recovers
automatically — the watchdog exists for *visibility*, not failover.

---

## Pairing

| Variable | Default | Notes |
|---|---|---|
| `PAIR_PORT` | `19500` | HTTP server port for `milvus-onprem pair`. Only used during initial deploy. |

The pair server uses a Bearer-token-authenticated HTTP server on this
port to distribute `cluster.env` to peers. Token is auto-generated per
session.

## Inspecting the live config

```bash
# show this node's role + cluster info:
./milvus-onprem status | head -10

# show the rendered config that's mounted into Milvus:
cat rendered/$(./milvus-onprem status 2>&1 | awk '/this node:/{print $3; exit}')/milvus.yaml
```

Or just `cat cluster.env` — it's intentionally small.

---

## Editing cluster.env after deploy

Most edits require:

```bash
./milvus-onprem render        # regenerates rendered/<node>/{compose,milvus.yaml,nginx.conf}
./milvus-onprem up            # picks up the new compose
```

Some changes need extra steps:

| Change | Extra action |
|---|---|
| `MINIO_ACCESS_KEY` / `MINIO_SECRET_KEY` | `teardown --full` + redeploy. MinIO can't rotate creds in distributed mode. |
| `MQ_TYPE` | `teardown --full` + redeploy. Collection metadata in etcd references MQ topics. |
| `MILVUS_IMAGE_TAG` (cross-major) | Plan a migration. Cross-major upgrades require milvus-backup → drop → restore. See [OPERATIONS.md](OPERATIONS.md). |
| `MILVUS_IMAGE_TAG` (same major.minor patch) | Just `render && up`; safe. |
| `PEER_IPS` | Tear down + redeploy. Re-numbering existing nodes mid-life is unsupported. |
| `WATCHDOG_*` | `sudo systemctl restart milvus-watchdog` (after `milvus-onprem install --with-watchdog`). |

---

## What's NOT in cluster.env

A few things are intentionally NOT in cluster.env, even though they're
configurable:

- **TLS certs** — not yet templated. All inter-node traffic is plain
  HTTP/gRPC.
- **RBAC users / passwords** — Milvus auth is disabled by default;
  edit `templates/<version>/milvus.yaml.tpl` and re-render to enable.
- **Milvus runtime knobs (segment size, GC tuning, etc.)** — these
  override Milvus defaults but aren't templated; either fork the
  template or mount an additional config file.

If you need any of these, open an issue describing the use case.
